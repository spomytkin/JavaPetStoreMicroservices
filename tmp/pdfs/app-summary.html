Java PetStore Microservices

Repo summary based on README, Maven modules, service configs, gateway routes, frontend code, and local Docker/Kubernetes assets.

What It Is

An educational PetStore app refactored from a Java EE style design into Spring Boot microservices with an Angular frontend. The repo packages business services, API gateway, auth/infrastructure configs, and observability tooling for local and Kubernetes-based environments.

Who It's For

Primary persona: Java and full stack developers learning or demonstrating how an e-commerce style application can be split into Spring Boot microservices with gateway, messaging, auth, and frontend layers.

What It Does

  • Shows products and lets users add new products through the Angular UI and product API.
  • Accepts orders through the gateway and order service.
  • Checks stock availability through the inventory service before saving an order.
  • Publishes order-placed events to Kafka after successful orders.
  • Consumes order events in the notification service and sends email confirmations.
  • Exposes aggregated Swagger docs through the API gateway.
  • Includes AI assistant chat endpoints in product service via LangChain4j and OpenAI config.

How To Run

  1. Start local infrastructure from repo root: docker compose up -d.
  2. Start the Angular app from frontend/: npm install, then npm start and open http://localhost:4200.
  3. Start backend services: explicit repo-wide command not found; repo evidence shows Spring Boot apps in api-gateway, product-service, order-service, inventory-service, and notification-service.
  4. Optional for assistant chat: set OPENAI_API_KEY for product-service. Exact assistant usage docs: Not found in repo.

How It Works

Flow: Angular frontend calls the API gateway at localhost:9000 for product and order requests. Gateway routes /api/product, /api/order, and /api/inventory to backing services and applies circuit breaker fallbacks.

Data: Product service uses MongoDB. Order and inventory services use MySQL with Flyway. Order service calls inventory synchronously to verify stock, then persists the order and publishes an Avro-backed Kafka event. Notification service listens to the Kafka topic and sends email. Keycloak and OIDC config support authentication; Grafana, Prometheus, Loki, and Tempo support observability.
